# Нейромережевий MPPT-трекер

## Принцип

Замість того, щоб шукати MPP методом проб і помилок (як P&O), нейромережа **передбачає** напругу MPP безпосередньо за виміряними фізичними параметрами:

$$ V_{mpp} = f_{NN}(G, T_{cell}) $$

де $f_{NN}$ — навчена повнозв'язна (fully-connected) мережа.

## Архітектура

```
   Input layer       Hidden 1         Hidden 2     Output
   (2 neurons)      (12 neurons)     (8 neurons)   (1 neuron)

   G/G_norm  ──┐                                    
               │      ┌──────┐       ┌──────┐    
               ├─────►│ tanh ├──────►│ tanh ├──────► V_mpp / Voc_arr
               │      └──────┘       └──────┘    
   T_norm   ──┘    (повнозв.)     (повнозв.)   (лінійний)
```

**Розміри ваг:**
- W₁: 12×2, b₁: 12×1
- W₂: 8×12, b₂: 8×1
- W₃: 1×8, b₃: 1×1
- Усього: 24+8+96+8+8+1 = **145 параметрів**.

## Прямий прохід — `nn_forward.m`

```
a₀ = x                                  (вхід)
z₁ = W₁ · a₀ + b₁;   a₁ = tanh(z₁)
z₂ = W₂ · a₁ + b₂;   a₂ = tanh(z₂)
z₃ = W₃ · a₂ + b₃;   y  = z₃           (лінійний вихід)
```

## Тренування — `nn_train.m`

**Алгоритм:** mini-batch SGD (Stochastic Gradient Descent) з:
- функцією втрат MSE: $L = \frac{1}{N}\sum (y_{pred} - y_{true})^2$;
- фіксованим learning rate (0.02);
- batch size = 64;
- 800 епох;
- train/val split = 80/20.

**Backpropagation** (вручну, без бібліотек):

Для вихідного шару (лінійного):
$$ \frac{\partial L}{\partial z_3} = \frac{2}{N}(a_3 - y_{true}) $$

Для прихованих шарів (з tanh):
$$ \frac{\partial L}{\partial z_i} = (W_{i+1}^T \cdot \frac{\partial L}{\partial z_{i+1}}) \odot (1 - \tanh^2(z_i)) $$

Градієнти ваг:
$$ \frac{\partial L}{\partial W_i} = \frac{\partial L}{\partial z_i} \cdot a_{i-1}^T $$
$$ \frac{\partial L}{\partial b_i} = \sum \frac{\partial L}{\partial z_i} $$

Оновлення ваг:
$$ W_i \leftarrow W_i - \eta \cdot \frac{\partial L}{\partial W_i} $$

де $\eta$ — learning rate.

## Генерація навчального набору — `nn_generate_dataset.m`

1. Генерується 3000 пар (G, T):
   - G ∈ [50, 1200] Вт/м² (логарифмічний розподіл — більше точок при низькій G);
   - T ∈ [0, 70] °C (рівномірний).
2. Для кожної пари обчислюється «справжня» V_mpp через `pv_mpp` (однодіодна модель + пошук argmax по сітці V).
3. На входи додається штучний шум (±2%), щоб мережа була стійкою до похибок сенсорів.
4. Дані нормалізуються: G/1200, (T-25)/50, V_mpp/Voc_array.

## Інференс у трекері — `mppt_nn.m`

На кожному кроці симуляції:
```matlab
x = [G_meas / G_norm; (T_meas - 25) / T_norm];
y = nn_forward(x, weights, cfg);
V_cmd = y * Voc_array;
```

Жодних збурень або ітерацій — **одне множення матриць і tanh** дає відразу робочу напругу.

## Переваги

1. **Миттєва реакція на зміну G, T** — не потрібно ітеративного сходження.
2. **Немає steady-state oscillation** — мережа видає одну напругу, а не «гойдає» її.
3. **Стійкість до drift** — при швидких хмарах не плутається, бо вимірює G безпосередньо.
4. **Приховує модель панелі у вагах** — мережа узагальнює, а не зберігає таблицю.

## Недоліки

1. **Точність обмежена якістю тренування**:
   - Проста мережа (12+8) має RMSE ~5% на V_mpp.
   - У стаціонарному режимі це дає трохи нижчу ефективність, ніж у стабільного P&O.
2. **Out-of-distribution**: якщо в експлуатації виникають (G, T), яких не було в тренувальному наборі, точність падає.
3. **Залежність від датчиків G, T** — потрібні фізичні сенсори, на відміну від P&O, який працює лише за вимірюванням P і V.
4. **Прив'язка до моделі панелі** — мережа навчена для KC200GT з конкретними параметрами; для іншої панелі треба перетреновувати.
5. **Вища обчислювальна вартість** — $\sim 2 \cdot 12 + 12 \cdot 8 + 8 \cdot 1 = 128$ операцій множення/додавання на крок проти ~5 операцій у P&O. На потужному мікроконтролері — ще нічого, але на дешевому 8-біт це може бути проблемою.

## Що показують експерименти

- **exp1_full_day** — типова робота NN, порівняння енергетичної віддачі за добу.
- **exp2_transient** — реакція NN на швидкі хмари (виграє у P&O).
- **exp3_sweep_cloudiness** — як NN тримає ефективність при високій мінливості.
- **exp5_compute_cost** — вимір реальної обчислювальної вартості.

## Можливі покращення (для розширення курсової)

1. **Ширша мережа** (16+16, 24+12) — точніше, але дорожче.
2. **Ще один вхід** — V_prev або dG/dt — зробить NN-трекер «вимірюваним» крім миттєвих величин.
3. **Гібрид NN+P&O** — мережа дає початкову точку, потім P&O донастроює (поєднує переваги).
4. **Перенавчання за поточних умов** — online learning, якщо панелі деградують з часом.
5. **Навчання на реальних даних** — замість синтетики від моделі.
