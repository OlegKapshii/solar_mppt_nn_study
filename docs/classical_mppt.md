# Класичний P&O Алгоритм MPPT

## Принцип роботи

MPPT (Maximum Power Point Tracking) - це алгоритм, що дозволяє підібрати опір навантаження сонячної панелі таким чином, щоб отримати максимальну потужність в будь-яких умовах освітленості та температури.

Алгоритм P&O (Perturb & Observe) базується на простому принципі:
- якщо потужність збільшилась, продовжити рух в тому ж напрямку
- якщо потужність зменшилась, розвернути у протилежну сторону

## Математичний опис

### Умови роботи

На I-V характеристиці панелі точка максимальної потужності знаходиться на:

$$\frac{dP}{dV} = 0$$

Тобто похідна потужності по напрузі дорівнює нулю.

### Алгоритм

```
На кожному кроці:

1. Виміряти P(k), V(k) - поточна потужність та напруга
2. Обчислити ΔP = P(k) - P(k-1), ΔV = V(k) - V(k-1)
3. Якщо ΔP == 0: напруга на оптимумі, залишити як є
4. Якщо ΔP > 0: рухатися в напрямку ΔV
   V(k+1) = V(k) + ΔV_step
5. Якщо ΔP < 0: рухатися в протилежному напрямку
   V(k+1) = V(k) - ΔV_step
```

### Математична реалізація

```
IF ΔP > 0:
    IF ΔV > 0:
        V_new = V + dV_step
    ELSE:
        V_new = V - dV_step
ELSE:
    IF ΔV > 0:
        V_new = V - dV_step
    ELSE:
        V_new = V + dV_step
```

## Параметри алгоритму в проекті

| Параметр | Значення | Опис |
|----------|----------|------|
| **dV_step** | 0.5 V | Крок зміни напруги |
| **Період оновлення** | 1 сек (базовий) | Як часто перераховується V |
| **Максимальна напруга** | 320 V | Робоча межа для масиву 10s x 2p |
| **Мінімальна напруга** | 30 V | Нижня робоча межа |

У проєкті також є адаптивний варіант P&O+Pyr (mppt_po_adaptive.m), де крок і реакція залежать від зміни освітленості.

## Переваги алгоритму

✅ **Простота реалізації** - лише кілька рядків коду  
✅ **Низька обчислювальна складність** - O(1) на крок  
✅ **Не потребує знання I-V характеристики** - адаптивний  
✅ **Стабільна робота** - не розходиться

## Недоліки алгоритму

❌ **Повільна реакція** на швидкі зміни освітленості  
❌ **Осциляції** навколо оптимальної напруги  
❌ **Залежність від dV** - великий крок дає осциляції, малий - повільну реакцію  
❌ **Неоптимальність при хмарах** - неправильно інтерпретує поведінку  

## Типові проблеми в умовах мінливої освітленості

### Проблема 1: Неправильна інтерпретація при раптовій зміні

Коли хмара прикриває панель:
- Освітленість падає різко (ΔG < 0, велике)
- Потужність падає (ΔP < 0)
- P&O розвертає напругу в протилежний напрямок
- АЛЕ це може бути неправильним напрямком!

### Проблема 2: Повільне слідування

При переході від глибокої хмари до яскравого сонця:
- Алгоритм рухається дуже повільно (крок 0.5V за 1 сек)
- На досягнення оптимуму може затратити 10-30 секунд
- Частково енергія втрачається

### Проблема 3: "Коливання навколо оптимуму"

В нестійких умовах алгоритм може:
- Коливатись навколо оптимуму
- Переходити від одного локального максимуму до іншого
- Витрачати енергію на переходи

## Модифікації базового алгоритму

В литературі існують вдосконалення P&O:

1. **Adaptive dV** - крок залежить від величини ΔP
2. **Incremental Conductance** - використання похідної I-V характеристики
3. **Fuzzy Logic MPPT** - вибір напрямку на основі нечіткої логіки
4. **Hybrid MPPT** - комбінація кількох методів

Але в проекті ми використовуємо **базову версію** для простоти.

## Порівняння з оптимальним управлінням

На практиці:
- **Оптимальна напруга** = унікальна V для кожної пари (G, T), де dP/dV = 0
- **P&O дає**: приблизну напругу, що коливається навколо оптимуму

Мета проекту - показати, як нейромережа може давати більш точну та швидку оцінку оптимальної напруги.

## Важлива примітка про втрати 10%

Фраза "P&O має втрати до 10%" не є загальним правилом.

- У базових сценаріях поточної моделі P&O має дуже високу ефективність (близько 99%+).
- Втрати порядку 10% виникають у спеціально ускладнених режимах (поєднання рідких оновлень, шуму, затримки, часткового затінення surrogate).

Тому оцінку втрат потрібно завжди давати разом із вказанням режиму тестування.
