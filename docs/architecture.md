# Архітектура системи

## Загальна схема

```
Модель сонячної радіації (Львів, день року)
         ↓
    Модель хмар (варіації освітленості)
         ↓
    Модель панелі KC200GT
    (I-V характеристика)
         ↓
    ┌────────────────────────┐
    │                        │
    ▼                        ▼
P&O MPPT алгоритм     Нейронна мережа MPPT
(Класичний)            (Натренована)
    │                        │
    └────────────────────────┘
         ↓
    Порівняння результатів
         ↓
    Аналіз метрик
```

## Модулі системи

### 1. Модель сонячної панелі (`solar_model/`)

**Файл**: `get_panel_characteristics.m`
- Параметри KC200GT (STC: 200W, 26.3V, 7.61A)
- Температурні коефіцієнти
- Обчислення I-V характеристики

**Файл**: `calculate_panel_output.m`
- Розрахунок струму панелі за освітленістю та температурою
- Динамічне визначення оптимальної напруги

**Файл**: `get_solar_irradiance.m`
- Модель сонячної радіації для Львова
- Залежність від часу дня та дня року
- Врахування кута сонця

### 2. Модель хмар (`cloud_model/`)

**Файл**: `apply_cloud_variation.m`
- Синтетичні варіації хмар
- Моделі: раптові зміни, плавні переходи, періодичні коливання

### 3. Класичний P&O алгоритм (`mppt_classical/`)

**Файл**: `mppt_po.m`
- Реалізація алгоритму P&O
- Входи: поточна напруга (V), поточна потужність (W), час симуляції
- Вихід: нова встановлена напруга
- Параметри: крок (dV = 0.5V), період оновлення (1 сек)

**Алгоритм**:
```
if P(t) > P(t-1):
    if V(t) > V(t-1):
        V(t+1) = V(t) + dV  # Крок в тому ж напрямку
    else:
        V(t+1) = V(t) - dV  # Крок в іншому напрямку
else:
    if V(t) > V(t-1):
        V(t+1) = V(t) - dV  # Розворот
    else:
        V(t+1) = V(t) + dV  # Розворот
```

### 4. Нейронна мережа (`neural_network/`)

**Архітектура**: Повнозв'язна мережа 2-8-4-1

**Файл**: `nn_init.m`
- Ініціалізація ваг (Xavier ініціалізація)
- Ініціалізація bias

**Файл**: `nn_forward.m`
- Пряма передача через мережу
- Входи: `[irradiance; temperature]`
- Вихід: `optimal_voltage`

**Файл**: `nn_train.m`
- Gradient descent з momentum
- Loss function: MSE
- Época: 100-200
- Learning rate: 0.01

**Параметри**:
- Входи (2): G (W/m²), T (°C)
- Прихований 1 (8): tanh активація
- Прихований 2 (4): tanh активація
- Вихід (1): лінійна активація

### 5. Генерація даних (`data_generation/`)

**Файл**: `generate_training_data.m`
- Синтетичні сценарії: 100+ комбінацій (G, T)
- Розраховує оптимальну напругу для кожної комбінації
- Генерує `.mat` файл для тренування

**Діапазони**:
- Освітленість: 50-1000 W/m²
- Температура: 15-60°C

### 6. Основні симулятори (`simulation/`)

**Файл**: `run_full_simulation.m`
- Симуляція системи на протязі обраного дня
- Можливість вибору сценарію хмар
- Виходи: часові ряди V, I, P для обох алгоритмів

**Файл**: `compare_algorithms.m`
- Паралельне порівняння P&O та NN MPPT
- Розрахунок метрик
- Експорт результатів

### 7. Аналіз результатів (`analysis/`)

**Файл**: `analyze_results.m`
- Розрахунок: енергія, ефективність, стабільність
- Статистичні показники

**Файл**: `plot_results.m`
- Графіки: Irradiance, Voltage, Current, Power
- Порівняльні графіки
- Аналіз особливостей

## Типові сценарії тестування

### Сценарій 1: Ясний день
- Плавна зміна освітленості від 0 до 1000 і назад
- Показує нестабільність P&O при нормальних умовах

### Сценарій 2: Хмарний день
- Раптові стрибки освітленості
- Демонструє повільність P&O та швидкість NN

### Сценарій 3: Частіші хмари
- Коливання освітленості кожні 2-3 хвилини
- Показує переваги NN у швидкості реакції

### Сценарій 4: Температурні варіації
- Комбінація змін освітленості та температури
- Показує адаптивність NN до обох параметрів

## Дані, які плином часу записуються

```
Для кожного часового кроку (dt = 1 сек):
├── Time (s)
├── Irradiance (W/m²)
├── Temperature (°C)
├── P&O MPPT:
│   ├── Voltage_PO (V)
│   ├── Current_PO (A)
│   └── Power_PO (W)
├── NN MPPT:
│   ├── Voltage_NN (V)
│   ├── Current_NN (A)
│   └── Power_NN (W)
└── Optimal (теоретичний максимум):
    └── Power_optimal (W)
```

## Вхідні параметри для варіювання

Можна змінювати:

1. **В P&O алгоритмі**:
   - Крок зміни напруги (dV)
   - Період оновлення

2. **В нейромережі**:
   - Архітектуру (кількість шарів, нейронів)
   - Learning rate при тренуванні
   - Функції активації

3. **В сценаріях**:
   - Інтенсивність хмар
   - Амплітуду темп. варіацій
   - Кількість панелей в масиві

## Очікувані висновки

1. P&O: Простий, але повільний, оптимальний для стабільної освітленості
2. NN: Швидкий, адаптивний, але потребує попередньої підготовки даних
3. Можлива комбінація: NN для швидкого наближення + P&O для fine-tuning
