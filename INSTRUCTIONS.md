# Інструкції

## Вступ

Цей проект порівнює два алгоритми відслідковування точки максимальної потужності (MPPT) сонячних панелей:

1. **Класичний P&O алгоритм** - простий і розповсюджений
2. **Нейромережевий MPPT** - новий підхід із застосуванням машинного навчання

Мета: зрозуміти, як нейронна мережа може покращити ефективність MPPT системи.

## Швидкий старт

### Крок 1: Запуск стартового скрипту

```matlab
START
```

Виберіть опцію 1: генерація даних для тренування

### Крок 2: Запуск однієї симуляції

Виберіть опцію 2: запуск `run_full_simulation`

Цей скрипт:
- Генерує сонячну радіацію для дня в Львові
- Застосовує варіації хмар
- Моделює температуру панелі
- Запускає P&O та NN MPPT алгоритми паралельно
- Виводить графіки та метрики

### Крок 3: Порівняння на різних сценаріях

Виберіть опцію 3: запуск `compare_algorithms`

Це запустить 6 різних сценаріїв:
- Ясний день
- Хмарний день (раптові зміни)
- Хмарний день (частісті хмари)
- Змішаний сценарій
- Зимовий день
- Весняний день

### Крок 4: Аналіз результатів

Виберіть опцію 4: запуск `analyze_results`

Це виведе детальний аналіз з таблицями та статистикою.

## Проведення власних досліджень

### Дослідження 1: Вплив кроку P&O на ефективність

**Гіпотеза**: Менший крок дає меншу ефективність через повільність, більший - осциляції

**Виконання**:
1. Відкрийте файл `mppt_classical/mppt_po.m`
2. Змініть `dV_step` з 0.5 на різні значення: 0.2, 0.5, 1.0, 2.0 V
3. Запустіть `run_full_simulation` для кожного значення
4. Запишіть енергію та помилку напруги

**Очікувані результати**:
- dV = 0.2 V: висока точність, але повільний відгук
- dV = 0.5 V: баланс (оптимум)
- dV = 1.0 V: швидкий, але осциляції
- dV = 2.0 V: великі осциляції

### Дослідження 2: Архітектура нейронної мережи

**Гіпотеза**: Більша мережа -> більша точність, але довше тренується

**Виконання**:
1. Відкрийте `neural_network/nn_init.m`
2. Змініть `network.layer_sizes` з [2, 8, 4, 1] на:
   - [2, 4, 2, 1] - мала мережа
   - [2, 16, 8, 1] - велика мережа
   - [2, 8, 8, 4, 1] - глибша мережа
3. Запустіть `generate_training_data()`
4. Запустіть `run_full_simulation`
5. Порівняйте результати

**Метрики для аналізу**:
- Точність (помилка від оптимуму)
- Час тренування
- Ефективність на тестових даних

### Дослідження 3: Вплив умов на різні алгоритми

**Гіпотеза**: P&O гірше при швидких змінах, NN одноманітно добрий

**Виконання**:
1. Запустіть `compare_algorithms`
2. Проаналізуйте результати для кожного сценарію
3. Заповніть таблицю:

```
Сценарій          | P&O Ефект | NN Ефект | Різниця (%)
──────────────────────────────────────────────────────
Ясний день        |    %      |    %     |     %
Раптові хмари     |    %      |    %     |     %
Частісті хмари    |    %      |    %     |     %
...
```

### Дослідження 4: Тренування мережі на складніших сценаріях

**Гіпотеза**: Якщо тренувати NN на даних з хмарами, вона краще адаптується

**Виконання**:
1. Створіть новий скрипт `advanced_training.m`
2. Генеруйте дані не просто з комбінацій (G, T), а з реальних часових рядів із хмарами
3. Порівняйте результати

### Дослідження 5: Гібридний підхід

**Гіпотеза**: Комбінація NN для грубого наближення + P&O для fine-tuning найкраща

**Виконання**:
1. Створіть `mppt_hybrid.m` що:
   - Використовує NN для швидкого наближення V_nn
   - Застосовує P&O з малим кроком навколо V_nn
2. Порівняйте з чистим P&O та чистим NN

## Структура звіту

Звіт повинен мати структуру:

1. **Вступ** (2-3 сторінки)
   - Описання проблеми MPPT

4. **Висновки** (2-3 сторінки)

## Часто задавані питання

### Q: Чому мережа не поліпшує результати?
A: Можливо:
- Недостатньо тренувальних даних
- Погана архітектура (занадто мала)
- Поганий learning rate
- Дані не охоплюють усі сценарії

### Q: Як змінити кількість тренувальних прикладів?
A: У `generate_training_data.m` змініть параметр:
```matlab
[training_data, validation_data] = generate_training_data(300);  % 300 замість 150
```

### Q: Як запустити на іншій локації?
A: У `get_solar_irradiance.m` змініть:
```matlab
latitude = 49.84;   % Ваша широта
longitude = 24.03;  % Ваша довгота
```

### Q: Як аналізувати окремий проміжок часу?
A: У `run_full_simulation.m` змініть:
```matlab
start_hour = 9;   % 9:00
end_hour = 12;    % 12:00 (3 години)
```
